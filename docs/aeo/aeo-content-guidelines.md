# AEO Content Guidelines for Marketing Context Engineering

**Version:** 1.0
**Date:** 2025-09-17
**Purpose:** Streamlined guidelines for optimizing content for Answer Engine Optimization (AEO)

---

## Overview

Answer Engine Optimization (AEO) is the evolution of SEO for AI-powered search engines like ChatGPT, Claude, Perplexity, and Gemini. These guidelines enable our existing marketing system to create content that dominates AI-generated answers while maintaining our Marketing Context Engineering approach.

---

## Core AEO Principles

### 1. **Direct Answer Priority**
- First 150 words must provide a clear, definitive answer
- Use declarative statements that AI engines can extract
- Include key technical details and unique positioning

### 2. **Comprehensive Depth**
- Target 2000+ words for pillar content (vs 1500 for traditional SEO)
- Address 10-15 related sub-questions within the content
- Provide implementation details and measurable outcomes

### 3. **Question-Based Structure**
- Build content around specific questions users ask AI engines
- Include FAQ sections with 10-15 relevant questions
- Address follow-up questions AI engines commonly generate

### 4. **Citation-Ready Content**
- Create content specifically designed to be cited by AI engines
- Include quotable insights and statistical data
- Maintain authority through technical depth and expertise

---

## Content Structure for AEO

### Template Structure
```
1. Direct Answer (150 words)
   - Clear response to primary question
   - Key benefits and unique value proposition
   - Technical credibility indicators

2. Comprehensive Context (500-800 words)
   - Business background and strategic importance
   - Competitive differentiation
   - Technical implementation details

3. Implementation Framework (600-1000 words)
   - Step-by-step methodology
   - Specific tools and approaches
   - Measurable outcomes and success metrics

4. FAQ Section (400-600 words)
   - 10-15 related questions and answers
   - Address common implementation concerns
   - Technical troubleshooting and optimization

5. Summary and Next Steps (200-300 words)
   - Key takeaways and action items
   - Clear call-to-action
   - Professional support offerings
```

### Enhanced Requirements by Content Type

**Blog Posts (AEO-Enhanced):**
- Minimum 2000 words (up from 1500)
- Direct answer in first paragraph
- FAQ section with 12+ questions
- Technical implementation details
- Citation-worthy statistics and insights

**Location Pages (AEO-Optimized):**
- Comprehensive local service descriptions
- FAQ addressing local concerns
- Technical expertise demonstrations
- Local citation opportunities

**Guides (AEO-Comprehensive):**
- 2500+ words for definitive coverage
- Multiple implementation scenarios
- Advanced troubleshooting sections
- Industry-specific applications

## Content Enhancement Process

### Using the AEO Enhancement Script

```bash
# Basic enhancement
./automation/aeo-enhance.sh content/blog-post.md

# Custom parameters
./automation/aeo-enhance.sh --target-words 2500 content/guide.md

# Dry run to preview changes
./automation/aeo-enhance.sh --dry-run content/location-page.md
```

### Manual Enhancement Checklist

**Pre-Enhancement:**
- [ ] Current word count documented
- [ ] Primary question identified
- [ ] Target sub-questions listed (10-15)
- [ ] Direct answer section planned

**Enhancement Process:**
- [ ] Add direct answer in first 150 words
- [ ] Expand content to target word count
- [ ] Generate comprehensive FAQ section
- [ ] Include technical implementation details
- [ ] Add citation-worthy insights and data

**Post-Enhancement:**
- [ ] Word count meets target (2000+ for pillars)
- [ ] FAQ section includes 10-15 questions
- [ ] Direct answer is clear and quotable
- [ ] Content addresses follow-up questions
- [ ] Technical depth demonstrates expertise

## Performance Measurement

### Key AEO Metrics

**Primary KPIs:**
- **Share of Voice**: Percentage of AI answers mentioning your brand
- **Citation Frequency**: Number of times cited across AI platforms
- **AI Traffic Conversion**: Conversion rates from AI engine traffic
- **Question Coverage**: Percentage of target questions optimized

**Platform-Specific Tracking:**
- **ChatGPT**: Monitor mentions in responses (averages 2.62 citations)
- **Perplexity**: Track citations (averages 6.61 citations)
- **Claude**: Monitor directory inclusion and mentions
- **Gemini**: Track mentions (averages 6.1 citations)

### Monthly AEO Review Process

1. **Performance Analysis**
   - Review AI engine visibility metrics
   - Analyze citation patterns and context
   - Compare AEO vs traditional SEO performance

2. **Content Optimization**
   - Identify highest-performing AEO content
   - Enhance underperforming pages
   - Expand successful question coverage

3. **Citation Building**
   - Monitor new citation opportunities
   - Build relationships with high-authority sources
   - Track competitive citation landscape

## Common Pitfalls to Avoid

### Content Issues
- **Thin Answers**: Direct answers under 100 words lack authority
- **Missing FAQs**: Content without FAQ sections misses citation opportunities
- **Generic Responses**: AI engines favor unique, specific insights
- **Insufficient Depth**: Under 2000 words fails to demonstrate comprehensive expertise

### Technical Issues
- **Poor Loading Speed**: AI engines favor fast-loading content
- **Missing Schema**: Structured data helps AI understanding
- **Broken Internal Links**: Poor site architecture reduces citation potential
- **Mobile Issues**: AI engines prioritize mobile-optimized content

### Strategic Issues
- **Inconsistent Positioning**: Conflicting messages across content
- **Competitive Neglect**: Failing to monitor competitor citation strategies
- **Platform Ignorance**: Not optimizing for specific AI engine preferences
- **Measurement Gaps**: Tracking traditional SEO metrics only
